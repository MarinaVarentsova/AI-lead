Стоп. Текущая логика БД взята из шаблона и НЕ соответствует нашему ТЗ.

Нельзя подгонять Supabase под шаблонные таблицы.
Нельзя пересоздавать нашу БД под serial/id integer.
Нельзя добавлять поля из шаблона.
Нельзя использовать свою схему Drizzle, если она расходится с фактической структурой Supabase.

Нужно переработать проект под утвержденную архитектуру БД.

Можно оставить:
- frontend UI;
- chat-widget;
- api-server;
- Vercel Function wrapper;
- KnowledgeBaseService;
- OpenAIService;
- общую структуру проекта.

Нужно снести/переписать:
- шаблонную Drizzle schema;
- шаблонную логику записи sessions/conversations/messages/diagnostic/leads;
- любые ожидания serial integer id;
- любые поля session_token, content, event_type, question_number, dict_id, is_custom, qualification_json, если их нет в утвержденной БД.

ВАЖНО:
Источник правды — существующая структура Supabase, созданная по нашему ТЗ.
Код должен подстроиться под БД, а не БД под код.

--------------------------------------------------
1. УТВЕРЖДЕННАЯ СТРУКТУРА БД
--------------------------------------------------

Все id — uuid.

Использовать только эти таблицы и поля.

--------------------------------------------------
ai_sessions
--------------------------------------------------

id uuid primary key default gen_random_uuid()
session_key text unique not null
first_page_url text
utm_source text
utm_medium text
utm_campaign text
utm_content text
utm_term text
created_at timestamptz default now()
updated_at timestamptz default now()

Назначение:
анонимная сессия посетителя сайта.

--------------------------------------------------
ai_conversations
--------------------------------------------------

id uuid primary key default gen_random_uuid()
session_id uuid references ai_sessions(id) on delete cascade
status text default 'started'
started_at timestamptz default now()
completed_at timestamptz
current_step text
created_at timestamptz default now()
updated_at timestamptz default now()

Назначение:
один диалог пользователя с AI.

--------------------------------------------------
ai_messages
--------------------------------------------------

id uuid primary key default gen_random_uuid()
conversation_id uuid references ai_conversations(id) on delete cascade
role text not null
message text not null
step text
created_at timestamptz default now()

Назначение:
вся переписка пользователя и ассистента.

ВАЖНО:
поле называется message, не content.

role:
- user
- assistant
- system

--------------------------------------------------
ai_diagnostic_answers
--------------------------------------------------

id uuid primary key default gen_random_uuid()
conversation_id uuid references ai_conversations(id) on delete cascade

experience_area text
experience_area_raw text

experience_years text
experience_years_raw text

education_type text
education_type_raw text

goal text
goal_raw text

created_at timestamptz default now()
updated_at timestamptz default now()

Назначение:
итоговые ответы диагностики одной записью на conversation_id.

ВАЖНО:
не хранить диагностику построчно через question_number.
У нас 4 ключевых ответа в одной строке.

--------------------------------------------------
ai_contacts
--------------------------------------------------

id uuid primary key default gen_random_uuid()
conversation_id uuid references ai_conversations(id) on delete cascade

name text
phone text
email text
telegram text
contact_channel text
preferred_time text
comment text

created_at timestamptz default now()

Назначение:
контактные данные клиента после диагностики.

--------------------------------------------------
ai_leads
--------------------------------------------------

id uuid primary key default gen_random_uuid()
conversation_id uuid references ai_conversations(id) on delete cascade
contact_id uuid references ai_contacts(id) on delete set null

lead_score integer
lead_temperature text
recommended_track text
recommended_tariff text

main_objection text

created_at timestamptz default now()

Назначение:
квалифицированный лид после диагностики / консультации.

ВАЖНО:
не использовать session_id в ai_leads.
Связь идет через conversation_id.
Если нужен session_id — получать через ai_conversations.session_id.

id uuid primary key default gen_random_uuid()

3. ОБЯЗАТЕЛЬНАЯ ЛОГИКА ЗАПИСИ ДАННЫХ
--------------------------------------------------

Сценарий должен работать так:

Шаг 1. Открытие виджета

При первом открытии или нажатии "Начать диагностику":

1. Создать ai_sessions:
- session_key = uuid/random token
- first_page_url = текущий URL, если доступен
- UTM поля, если есть

2. Вернуть frontend:
{
  sessionId: ai_sessions.id,
  sessionKey: ai_sessions.session_key
}

Не использовать session_token.

--------------------------------------------------
Шаг 2. Старт диагностики

После создания сессии создать ai_conversations:

- session_id = ai_sessions.id
- status = 'diagnostic_in_progress'
- current_step = 'experience_area'

Вернуть frontend:
{
  conversationId: ai_conversations.id,
  currentStep: 'experience_area'
}

--------------------------------------------------
Шаг 3. Сообщения

Каждый вопрос ассистента и каждый ответ пользователя писать в ai_messages:

- conversation_id
- role = 'assistant' или 'user'
- message = текст
- step = текущий шаг

Не использовать поле content.

--------------------------------------------------
Шаг 4. Ответы диагностики

Диагностика состоит из 4 вопросов:

1. experience_area
2. experience_years
3. education_type
4. goal

Ответы сохранять в ai_diagnostic_answers одной записью на conversation_id.

При выборе из справочника:
- основное поле = code справочника
- raw поле = отображаемый текст или исходный ввод пользователя

Пример:

experience_area = 'construction'
experience_area_raw = 'строительство / ремонт'

Если пользователь ввел свой текст:
- основное поле = 'other' или 'need_clarification' по смыслу
- raw поле = точный текст пользователя

После каждого ответа:
- обновлять текущую запись ai_diagnostic_answers
- обновлять ai_conversations.current_step

После 4-го ответа:
- ai_conversations.status = 'diagnostic_completed'
- current_step = 'diagnostic_completed'

--------------------------------------------------
Шаг 5. Предварительная рекомендация

После завершения 4 вопросов:

1. Использовать knowledge_base_inobr_ai_consultant_v2.md.
2. Сформировать предварительный вывод пользователю.
3. Не обещать заработок, трудоустройство, судебный результат, скидку.
4. Если education_type = 'school_only':
   - не предлагать ССТЭ / ДПО
   - рекомендовать приемку квартир
   - в будущий ai_brief добавить пометку:
     "У клиента только аттестат. ССТЭ / ДПО не предлагать как основной вариант."

--------------------------------------------------
Шаг 6. AI-консультация

После диагностики пользователь может задавать вопросы.

Каждое сообщение писать в ai_messages.

AI отвечает только по базе знаний:
knowledge_base_inobr_ai_consultant_v2.md

Если ответа нет:
"В моей базе нет точной информации по этому вопросу. Я передам его менеджеру, чтобы он ответил корректно."

--------------------------------------------------
Шаг 7. Сбор контакта

Контакт не собирать сразу.

Контакт собирать после:
- диагностики;
- предварительного вывода;
- 2–4 ответов на вопросы пользователя.

Когда пользователь выбрал канал связи и оставил данные:

создать ai_contacts:

- conversation_id
- name
- phone
- email
- telegram
- contact_channel
- preferred_time
- comment

2. Рассчитать lead_score.

5. Записать ai_leads:

--------------------------------------------------
4. LEAD SCORE
--------------------------------------------------

OpenAI не считает итоговый lead_score.
Lead score считает backend.

Правила:

+20 если education_type = higher_technical, secondary_technical, non_profile

+20 если education_type = higher_technical или secondary_technical

+20 если experience_years = from_3_to_10 или more_than_10

+20 если goal = extra_income, new_profession, expand_services, construction_expertise

+20 если installment_interest = true
или main_question связан со стоимостью / рассрочкой / договором / документами

+20 если start_readiness = now или this_month

+10 если contact_channel = phone, whatsapp, telegram, max

Ограничение:
если education_type = school_only:
- recommended_track = apartment_acceptance
- recommended_tariff = not_defined
- lead_score не выше 30
- ai_brief должен содержать пометку про аттестат

lead_temperature:

90+ = hot
60–89 = warm
30–59 = cold
0–29 = info
